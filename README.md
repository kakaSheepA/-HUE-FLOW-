# 色彩流动 HueFlow

## About

`色彩流动 HueFlow` 是一个面向 MasterGo 的配色与渐变插件，帮助你在设计过程中更快完成选色、调色、生成与应用这几类高频操作。

它把常用的颜色工作流整理成多个标签页，适合快速换色、统一色阶、生成弥散渐变、套用预设方案，也适合在选中图层后直接实时调整效果。

如果你经常在设计里来回切换颜色、渐变和背景氛围，这个插件可以把这些动作收敛到一个更顺手的工作台里。

### 一般工作流

- 想快速改颜色：去 `超级调色板` 或 `梯度色阶`。
- 想精细改一个颜色：去 `色彩调节`。
- 想做更有氛围的柔和大色块：去 `弥散渐变`。
- 想直接套用现成方案：去 `预设渐变`。

### 选中图层的意义

- 大多数操作都需要先选中图层，插件才知道要改哪个对象。
- `超级调色板`、`梯度色阶` 的点击会直接把颜色写进当前选区。
- `弥散渐变` 会根据当前选区实时同步，点击 `应用` 后再把最终效果写回去。

### 色彩调节

`色彩调节` 适合做细微改色，比如统一某个品牌色、微调高亮和阴影。

1. 打开 `色彩调节`。
2. 找到当前颜色对应的 HSB 轨道。
3. 拖动滑块观察颜色变化。
4. 如果只想改填充，可以关闭描边；如果想整体一起变，可以同时作用于描边。
5. 颜色会直接作用到当前选中的图层。

### 超级调色板

`超级调色板` 适合快速从一组现成色里挑一个颜色替换当前对象。

1. 打开 `超级调色板`。
2. 在右侧色块中找到你要的颜色。
3. 点击色块后，颜色会直接应用到当前选中的形状。
4. 如果你只是想拿色值，也可以直接点击复制。
5. 这个 tab 更适合“快速换色”，不是做复杂渐变。

### 梯度色阶

`梯度色阶` 适合拿标准化色阶，比如灰阶、品牌主色阶，快速套到 UI 设计里。

1. 打开 `梯度色阶`。
2. 先在左侧选择色阶家族，例如中性灰、橙、蓝等。
3. 再在右侧挑选某一档色阶。
4. 点击后会把对应颜色应用到当前选中的图层。
5. 中性灰的白到黑阶已经按常用透明度命名和展示，方便直接拿去用。
6. 如果你在做组件状态、层级、阴影层次，这个 tab 会比较顺手。

### 弥散渐变

`弥散渐变` 适合做大面积背景、氛围面板、海报底图这类柔和的 mesh 效果。

1. 打开 `弥散渐变`。
2. 页面上方会先显示当前预览。
3. 下方每个色块代表一个弥散锚点的颜色。
4. 点击色块后，可以直接用浏览器原生颜色选择器修改它。
5. 选中某个色块后，下方的滑块会控制这个锚点的参数：
   - 扩散：决定色团大小
   - 模糊：决定边缘柔和程度
   - 透明度：决定锚点显色强弱
   - 横向：控制位置左右移动
   - 纵向：控制位置上下移动
   - 噪点：让整体更自然一点
6. 调整时预览会实时变化。
7. 如果当前已经选中了图层，调整滑块、改颜色、随机生成都会实时同步到选区。
8. 想把当前结果固定到图层上时，点击 `应用`。
9. `随机生成` 会基于当前色板生成一版更协调的变体，适合快速试效果。

### 预设渐变

`预设渐变` 适合直接套用已经整理好的经典渐变，不需要自己从头配。

1. 打开 `预设渐变`。
2. 浏览不同的预设样式。
3. 点击任意一个预设，就会直接应用到当前选中对象。
4. 如果你只是想快速找一个方向感，这个 tab 会比手动调更快。

### 小提示

- 如果没有先选中图层，很多“应用”类操作不会有可见结果。
- `弥散渐变` 里的“预览”是参考结果，真正写入图层的是点击 `应用` 后的效果。
- `随机生成` 适合快速找氛围，不适合精确复刻某个设计稿。


## 常见问题

### 为什么我点了按钮没有变化？

通常是因为没有先选中图层。这个插件里大多数“应用”类操作都需要当前画布里有一个目标对象。

### 为什么弥散渐变看起来和应用到形状后的效果不完全一样？

现在插件已经尽量让预览和应用结果使用同一套参数，但不同尺寸、不同形状比例、不同容器背景都会让视觉结果有细微差异。

### 为什么有些颜色会直接改到选区，有些只是预览？

像 `超级调色板`、`梯度色阶` 这类功能是直接改当前选区的颜色；`弥散渐变` 则更偏向生成一个完整的背景效果，再通过 `应用` 写回图层。

### 随机生成出来的颜色为什么有时候很接近？

这是为了保证整体更和谐。当前随机逻辑更偏向同类色和邻近色，而不是完全无规则地乱跳。

### 我能不能把弥散渐变直接导出成图片？

当前 `弥散渐变` 的重点是把颜色效果应用到形状上。若你需要导出图片，建议先把效果应用到目标图层，再使用 MasterGo 自身的导出能力。

```bash
npm run build
```

## 项目结构

- `ui/`：插件界面
- `lib/`：插件主线程逻辑
- `messages/`：UI 和主线程通信协议
- `dist/`：构建产物

## 说明

- 本项目面向 MasterGo `canvas` 环境。
- `弥散渐变` 目前以“应用到所选形状”为主，预览和实际应用使用同一套参数。
- 代码会持续模块化拆分，界面逻辑和样式分别放在独立文件中维护。
